# code2llm — refactoring prompt for sumd project
# Generated: 2026-04-19
# Goal: profile-aware, on-demand generation of only the analysis files
#       that each profile actually embeds, so that sumd / sumr always
#       include fresh data without running the full --analyze pipeline.

## Context

The sumd tool generates SUMD.md (documentation) and SUMR.md (refactoring
report) from project sources.  Before rendering it calls `extract_project_analysis()`
which reads pre-existing files from `project/` on disk.

Currently:
- `map.toon.yaml`  → auto-regenerated by `_refresh_map_toon()` in pipeline.py (pure-Python, fast)
- `calls.toon.yaml`, `analysis.toon.yaml`, `duplication.toon.yaml`,
  `evolution.toon.yaml`, `validation.toon.yaml`
  → only generated when the user explicitly runs `sumd analyze .` or `sumd scan . --analyze`
  → these files may be stale or missing when sumd / sumr is run

The desired behaviour:
- `sumd scan .` (profiles: minimal / light / rich) → generate only the files
  that the `rich` profile embeds, right before embedding
- `sumr .` (profile: refactor) → generate only the files that the `refactor`
  profile embeds, right before embedding

  project/map.toon.yaml         ← `sumd map .`          (built-in, pure-Python)
  project/calls.toon.yaml       ← `code2llm ./ -f toon -o project/`
  project/analysis.toon.yaml    ← `code2llm ./ -f toon -o project/`  (same run)
  project/evolution.toon.yaml   ← `code2llm ./ -f toon -o project/`  (same run)
  project/duplication.toon.yaml ← `redup scan . --format toon --output project/`
  project/validation.toon.yaml  ← `vallm batch . --recursive --format toon --output project/`

(code2llm generates calls.toon, analysis.toon and evolution.toon in a single
invocation; the three are not separate runs.)

### Task 2 — `_refresh_analysis_files()` in `sumd/pipeline.py`

Add a function that runs only the tools required by the active profile:

```python
# sumd/pipeline.py  (add after _refresh_map_toon)

def _refresh_analysis_files(proj_dir: Path, profile: str) -> None:
    """Run only the external tools required for the given profile.

    Silently skips any tool not installed in .sumd-tools/venv.
    profile: 'minimal' | 'light' | 'rich' | 'refactor'
    """
    from sumd.extractor import required_tools_for_profile

    tools_needed = required_tools_for_profile(profile)
    if not tools_needed:
        return

    tools_dir = proj_dir / ".sumd-tools"
    venv_dir = tools_dir / "venv"
    bin_dir = venv_dir / "bin"
    if not bin_dir.exists():
        bin_dir = venv_dir / "Scripts"
    if not bin_dir.exists():
        return  # tools not installed — skip silently

    project_output = proj_dir / "project"
    project_output.mkdir(parents=True, exist_ok=True)

    def _exe(name: str):
        p = bin_dir / name
        if not p.exists():
            p = bin_dir / f"{name}.exe"
        return p if p.exists() else None

    # code2llm generates calls.toon, analysis.toon, evolution.toon in one run
    if "code2llm" in tools_needed:
        exe = _exe("code2llm")
        if exe:
            subprocess.run(
                [str(exe), "./", "-f", "toon", "-o", str(project_output), "--no-chunk"],
                capture_output=True, cwd=str(proj_dir),
            )

    if "redup" in tools_needed:
        exe = _exe("redup")
        if exe:
            subprocess.run(
                [str(exe), "scan", str(proj_dir),
                 "--format", "toon", "--output", str(project_output)],
                capture_output=True, cwd=str(proj_dir),
            )

    if "vallm" in tools_needed:
        exe = _exe("vallm")
        if exe:
            subprocess.run(
                [str(exe), "batch", str(proj_dir), "--recursive",
                 "--format", "toon", "--output", str(project_output)],
                capture_output=True, cwd=str(proj_dir),
            )
```

## Acceptance criteria

- `sumd .` on a project with `.sumd-tools/venv` + code2llm:
  SUMD.md embeds a *fresh* `calls.toon.yaml` automatically

- `sumr .` on the same project:
  SUMR.md embeds fresh `calls.toon.yaml`, `analysis.toon.yaml`,
  `duplication.toon.yaml`, `evolution.toon.yaml`, `validation.toon.yaml`

- On a project *without* `.sumd-tools/venv`:
  `sumd .` and `sumr .` complete without error; embed previous files
  from `project/` if they exist, or skip if they don't

- `sumd scan . --analyze` still works and doesn't double-run tools
